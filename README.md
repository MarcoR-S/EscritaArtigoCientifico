# 📄 artigoparaentrega_checkpoint3

**Entrega oficial do Checkpoint 3**

Este repositório foi desenvolvido para centralizar todos os artefatos relacionados à entrega final do artigo científico referente ao **Checkpoint 3**, incluindo a versão final do trabalho, os materiais de apoio utilizados durante a elaboração, o pipeline de geração de dados e os mecanismos de validação do processo de autoria.

> **Objetivo do repositório:** fornecer ao avaliador acesso organizado à documentação, aos dados utilizados no projeto e às evidências que comprovam o desenvolvimento gradual e autêntico do artigo.

# 📝 Pasta: `escrita/`

Esta pasta contém todo o material relacionado à produção textual do artigo.

Conteúdo esperado:

- **Versão final** do artigo entregue para avaliação;
- **Rascunhos intermediários** produzidos ao longo do desenvolvimento do projeto;
- Registros da evolução da escrita durante as diferentes etapas de elaboração.

O objetivo desta pasta é permitir a visualização da progressão do trabalho, desde suas versões preliminares até a versão consolidada submetida para avaliação.

---

# ⚙️ Pasta: `gerar dados/`

Esta pasta contém o notebook utilizado para geração e validação automatizada dos dados empregados no projeto.

O arquivo está disponibilizado no formato **Google Colab (`.ipynb`)**.

## Como executar

1. Abra o arquivo `.ipynb` utilizando o **Google Colab**;
2. Aguarde o carregamento completo do ambiente;
3. Execute todas as células do notebook:
   - Menu: **Runtime → Run all**; ou
   - Atalho: **Ctrl + F9**;
4. Aguarde a finalização da execução;
5. Os dados e validações serão gerados automaticamente pelo pipeline implementado.

> **Observação:** a execução integral do notebook permite reproduzir o processo de geração dos dados utilizados no artigo, garantindo transparência e reprodutibilidade dos resultados apresentados.

# 🔗 Pasta: `links/`

Esta pasta possui papel fundamental no processo de validação da autoria do trabalho.

Nela estão armazenados os links para as plataformas e ambientes onde as partes escritas do projeto foram desenvolvidas ao longo do semestre.

O propósito desta documentação é permitir que o avaliador consulte:

- Histórico de alterações;
- Controle de versões;
- Datas de criação e modificação;
- Evolução temporal dos documentos;
- Registro das contribuições realizadas durante o desenvolvimento do artigo.

> **Importante:** os materiais disponibilizados nesta pasta constituem evidências auditáveis do processo de escrita, demonstrando a construção gradual e orgânica do trabalho ao longo do projeto.

> A análise do histórico de versões permite verificar de forma transparente que o artigo foi desenvolvido por meio de sucessivas revisões e refinamentos, não sendo um conteúdo integralmente gerado por Inteligência Artificial.

# ✅ Orientação para Avaliação

Para uma análise completa do projeto, recomenda-se a seguinte sequência:

1. Consultar a pasta **`escrita/`** para visualizar o artigo final e seus rascunhos;
2. Executar o notebook presente em **`gerar dados/`** para reproduzir o processo de geração dos dados;
3. Acessar os documentos e históricos disponibilizados em **`links/`** para verificar a evolução temporal da produção textual.
